# 고래상사 디자인 시스템

> 회사 역할극 버추얼 크루 "고래상사" 공식 홈페이지의 디자인 기준.
> 색·타이포·간격·컴포넌트·이펙트·모션·레이아웃을 정리해, **누구든 같은 스타일의 페이지를 만들 수 있게** 하는 문서입니다.
> 테마 한 줄 요약: **다크 · 글래스 · 네온** — 짙은 네이비 캔버스 위 브랜드 블루 + 틸 액센트.

---

## 0. 디자인 원칙

새 화면을 만들 때 의사결정의 기준. 막히면 여기로 돌아온다.

| 원칙 | 내용 |
|---|---|
| 🌑 **어둠이 기본** | 짙은 네이비 캔버스를 바탕으로. 색은 절제하고, 빛나는 포인트로 시선을 모은다. |
| 💎 **겹쳐서 깊이** | 반투명 면·선·블러를 층층이 쌓아 평평하지 않은 입체감을 만든다. |
| ⚡ **큼직하고 단단하게** | 제목은 900 굵기에 좁은 자간. 영문은 윤곽선으로 크게 써 리듬을 준다. |
| 🌊 **움직임은 거들 뿐** | 스크롤 리빌이 골격. 점멸·셔머는 강조에만, 모션이 콘텐츠를 가리지 않게. |


## 2. 타이포그래피

전 구간 **Pretendard**. 큰 제목일수록 자간을 좁혀 단단하게. 한글은 `word-break:keep-all`로 단어 단위 줄바꿈.

### 타입 스케일

| 역할 | 크기 | 굵기 / 자간 |
|---|---|---|

| Section H2 | `clamp(42px,6vw,88px)` | 900 · 윤곽선 가능 |

| Label / Caption | `12–13px` | 800 · 대문자 ls `.2em` |
| Mono / Code | `12–14px` | JetBrains Mono 400–700 |

### 굵기

400 Regular · 500 Medium · 600 SemiBold · 700 Bold · 800 ExtraBold · 900 Black
(주로 **900**(헤드라인)·**800**(라벨)·**700**(버튼/강조)·**400**(본문)을 쓴다.)

## 6. 컴포넌트

### 버튼

굵기 700, 라운드 11px. Primary는 브랜드 블루에 발광 그림자. 호버 시 1px 떠오르고 보더가 브랜드색으로.

```css
.btn {
  display: inline-flex; align-items: center; gap: 8px;
  font-weight: 700; font-size: 15px; padding: 12px 20px;
  border-radius: 11px; border: 1px solid var(--line-2);
  background: var(--surface-2); color: var(--ink);
  cursor: pointer; transition: .15s;
}
.btn:hover    { border-color: var(--brand); transform: translateY(-1px); }
.btn.primary  { background: var(--brand); color: #fff;
                box-shadow: 0 8px 26px rgba(47,99,255,.4); }
.btn.sm       { padding: 8px 14px; font-size: 14px; }
.btn.ghost    { background: transparent; border-color: transparent; color: var(--ink-2); }
```

변형: **Primary / Default / Small / Ghost / Disabled**(투명도↓, `cursor:not-allowed`).

### 폼 & 인풋

반투명 면 + 1px 라인. 포커스 시 보더가 브랜드색으로, 면이 한 단계 밝아진다. 라벨은 600 굵기 보조색.

```css
.field input {
  width: 100%; font-size: 15px; padding: 11px 13px;
  border: 1px solid var(--line-2); border-radius: 10px;
  background: rgba(255,255,255,.04); color: var(--ink);
}
.field input:focus {
  outline: none; border-color: var(--brand);
  background: rgba(255,255,255,.07);
}
.field > span { font-size: 13px; font-weight: 600; color: var(--ink-2); }
```

상태: 기본 / 포커스(브랜드 보더) / 오류(`--hot` 보더 + 메시지).

### 배지 · 태그 · 라벨

- **라이브 배지** — 알약형, 점멸하는 초록 점(`#34d399`, `pulse 1.8s`)
- **부서 태그** — `rgba(255,255,255,.14)` 알약, 11px 700
- **섹션 라벨** — 12px 800, 대문자, `letter-spacing:.2em`, 브랜드색
- **카드 영문(EN)** — 액센트색, 대문자 `letter-spacing:.1em`
- **날짜 배지** — 브랜드 블루 채움, 라운드 8px
- **재생 배지** — `#ff0033` 채움

### 카드

네 가지 기본형. 카드 위에서는 살짝 떠오르고(`translateY(-4px)`) 보더가 밝아진다.

1. **비즈니스 그라데이션 카드** — `aspect-ratio:4/5`, 라운드 18px, 부서 그라데이션 배경 + 윤곽선 번호 + 영문 라벨 + 국문 제목. 5종 그라데이션을 순환.
   ```
   01 Live Streaming  linear-gradient(135deg,#0b2e8f,#2f6bff,#0a44cc)  📡
   02 Management      linear-gradient(135deg,#06564a,#10e0a6,#067a5e)  🐋
   03 Content         linear-gradient(135deg,#3a0fa6,#9333ff,#5a14c0)  🎬
   04 Commerce        linear-gradient(135deg,#7a4a00,#ffb220,#9a3800)  🛒
   05 Community       linear-gradient(135deg,#7a0a3c,#ff3d92,#a80a44)  🌊
   ```
2. **멤버 카드** — `aspect-ratio:3/4`, 라운드 12px, 부서 그라데이션(또는 사진) + 하단 스크림 + 이름·직책·부서 태그.
3. **정보 패널** — `--surface` 면, 라운드 16px, 제목 + 상단 구분선 행들 + 하단 액션 버튼.
4. **영상 카드 16:9** — 라운드 16px, `--shadow`, 중앙 `#ff0033` 재생 버튼, 하단 제목.

### 리스트 & 로우

상단 구분선(`--line`)으로 행을 나눈다. 호버 시 텍스트가 밝아지며 왼쪽으로 살짝 밀려나고(`padding-left:7px`), 화살표가 액센트색으로 점등.

- **클립 리스트** — 제목 + `↗`, active 행은 `--ink`
- **일정 로우** — 날짜 배지 + 설명(`<b>`로 시간 강조)
- **아카이브 로우** — 제목/부제 + "바로가기" 버튼

### 내비게이션

- **헤더** — 투명 → 스크롤 시 `rgba(7,10,20,.88)` + `blur(14px)` + 그림자로 굳음(`.scrolled`).
- **로고** — 21px 900, "고래" + "상사"(액센트색).
- **햄버거** — 44×44 면, 3선. 열리면 X자로 회전(`aria-expanded`).
- **드로어 메뉴** — 우측 슬라이드(`translateX(100%)→0`, `width:min(460px,84vw)`), 큰 타이포(`clamp(24px,4.4vw,40px)` 900), 번호 + 라벨.

### 모달 & 오버레이

블러 처리된 어두운 백드롭(`rgba(3,6,14,.82)` + blur) 위로 모달이 살짝 떠오르며 등장(`modalIn .28s`).

- **멤버 프로필 모달** — 2단 그리드(`1fr 1.4fr`): 아바타 + 정보. 부서 라벨·이름·직책·소개·스탯 3칸(카운트업)·이전/다음. ≤640px에서 단일 컬럼.
- **로그인 모달** — 단일 컬럼, `max-width:380px`. 제목·서브 + 아이디/비밀번호 + Primary 버튼.

### 섹션 헤더 패턴

1. **윤곽선 영문** — 작은 브랜드 라벨 + 대형 `-webkit-text-stroke` 영문 타이틀 + 서브카피. (예: BUSINESS)
2. **국문 + 액센트** — `크루 & <em>멤버</em>`처럼 강조어만 액센트색.

## 7. 모션

기본 이징 `cubic-bezier(.22,.61,.36,1)`. 스크롤 리빌이 골격, 점멸·셔머·플로팅은 강조 요소에만 절제해서.

| 모션 | 사양 | 용도 |
|---|---|---|
| 스크롤 리빌 | `translateY(40→0)` + opacity, 1s | 진입 시 떠오름(기본) |
| 카드 호버 | `translateY(-1~4px)` + 보더 전환 | 떠오르며 강조 |
| 펄스 | box-shadow ring 1.8s | 라이브 점 점멸 |
| 셔머 | background-position 6s linear | 그라데이션 텍스트 |
| 플로팅 오브 | float 16~21s ease-in-out | 배경 광원 부유 |
| 바운스 | `translateY` 2s | 스크롤 인디케이터 |
| modalIn | `translateY(18px) scale(.97)→0` .28s | 모달 등장 |

### 리빌 방향 유틸리티

```
.bottom-top   transform: translate(0, 100px)    /* 아래→위 */
.bottom-top0  transform: translate(0, 180px)
.left-right   transform: translate(-100px, 0)   /* 왼→오 */
.right-left   transform: translate(100px, 0)    /* 오→왼 */
.img-aniload  transform: translate(0,0); opacity: 1   /* 활성 상태 */
.tran01~05    transition-delay: .5s ~ 2.5s       /* 순차 지연 */
```

스크롤 위치가 요소 상단 +200px를 지나면 `.img-aniload`를 붙여 재생.

## 8. 가이드라인 (Do / Don't)

| ✅ DO | ❌ DON'T |
|---|---|
| 브랜드 블루는 주요 액션 한 곳에만 강하게 | 여러 버튼에 Primary를 남발하지 않기 |
| 큰 제목엔 좁은 자간(`-.03~-.045em`) | 본문까지 자간을 좁혀 가독성 해치지 않기 |
| 카드 위 텍스트엔 항상 스크림을 깔기 | 밝은 이미지 위에 흰 글씨를 맨몸으로 올리지 않기 |
| 섹션 사이는 넉넉히(80–140px) 띄우기 | 블록을 빽빽하게 붙여 답답하게 만들지 않기 |

## 9. 접근성

- **모션 줄이기** — `prefers-reduced-motion` 존중. 인트로·리빌·플로팅을 모두 끄고 즉시 표시.
- **포커스 가시성** — 인풋·버튼은 포커스 시 브랜드색 보더로 명확히 표시.
- **시맨틱 & ARIA** — 모달 `role="dialog"` + `aria-modal`, 햄버거 `aria-expanded`, 장식 요소 `aria-hidden`.
- **터치 타깃** — 버튼·햄버거는 최소 44px. 작은 화살표도 충분한 패딩 확보.

## 10. 페이지 레이아웃 청사진

원본 홈페이지의 섹션 구성 순서. 같은 페이지를 재현하려면 이 골격에 콘텐츠를 채운다.

```
헤더(고정)
 └ 로고 · 로그인 · 햄버거 → 우측 드로어 메뉴

SEC01  풀스크린 히어로
 └ 배경(오브 2개 + 도트그리드 + 그라데이션) · 거대 윤곽선 워드마크

 └ 배지 · H1(셔머 강조) · 본문 · 버튼 2개 · 스크롤 인디케이터

SEC02  BUSINESS — 가로 스크롤 핀
 └ 섹션 헤더(윤곽선 영문 BUSINESS)
 └ 비즈니스 카드 5장이 가로로 흐름(스크롤 → 가로 이동, GSAP pin+scrub)
 └ 카드별 폭·높이·세로 위치 차등 → 들쭉날쭉한 리듬

SEC03  크루 & 멤버
 └ 국문 헤더 + "전체 보기" 링크

푸터
 └ 로고 · 링크 버튼들 · 카피라이트
```

### 인터랙션 사양 요약

- **가로 스크롤 핀** — `#sectionPin`을 pin, `.slide-cont`를 `x: -(scrollWidth - innerWidth)`로 이동. `scrub:true`, `ease:'none'`. 진행도에 따라 카드 `.on` 순차 활성화. `≤989px`에서는 세로 스택.
- **인트로** — 로그인/리빌 잠금 3.6s. 700ms 로고 등장 → 2000ms 위로 사라짐 → 4000ms 히어로 표시. `sessionStorage`로 세션당 1회.
- **멤버 슬라이더** — Swiper 8, `slidesPerView:'auto'`, `spaceBetween:20`, `freeMode`, 좌우 버튼 네비.
- **스태거** — 멤버 카드마다 다른 `yPercent`(±12~56)로 스크롤 연동 이동.
- 모두 데스크톱(`>989px`) + 모션 허용 시에만. 그 외엔 즉시 정적 표시.

## 11. 바로 시작하기

1. `<head>`에 Pretendard를 불러온다.
   ```html
   <link href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.css" rel="stylesheet">
   ```

   ```css
   :root{
     --bg:#070a14;
     --surface:rgba(255,255,255,.045);
     --surface-2:rgba(255,255,255,.06);
     --ink:#eef2fb;
     --ink-2:#9aa6bd;
     --line:rgba(255,255,255,.10);
     --line-2:rgba(255,255,255,.16);
     --brand:#2f63ff;
     --brand-2:#1f4fd8;
     --accent:#0fb5b0;
     --hot:#ff4d4d; --warn:#ffcb45;
     --radius:16px;
     --shadow:0 10px 40px rgba(0,0,0,.45);
     --maxw:1360px;
   ```
